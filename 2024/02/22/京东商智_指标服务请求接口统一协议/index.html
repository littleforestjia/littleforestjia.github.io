1.2 请求入参

接口：com.jd.bdaa.arch.dataservice.UnionDataService

方法：fetchBizData

参数名 类型 必须 描述
header UReqData.Header 是 请求头
-appkey String 是 原子服务对调用方的授权key，jdos应用级标识
-uuid String 是 调用UUID，为了便于日志采集，需满足正则 [a-zA-Z0-9-|_.]+
-context HashMap<String, String> 是 放置不影响数据结果的参数，如统计、分流、鉴权等服务条件。业务如有诉求，经指标服务配置后可在链路中传递，否则会被拦截。目前可传递字段为：指标类型（不传默认离线）:key:INDEX_FREQ value:”REALTIME”/“OFFLINE” (实时/离线)资源场景标识Key（必传）:key:RES_APPKEYvalue:例”ge_jdv_boss”备注：当value为”MOCKER“时，代表使用数据探查能力，仅对入参做校验，不实际请求数据库。
body UReqData.Body 是 请求体
-indicators List 是 查询指标code列表如需动态传递修饰和函数，请查看本文档附录1
-attributes List 否 查询维度code列表
-criteria UCriteria 否 查询条件
– criterions List 否 筛选过滤查询条件，不支持“或”逻辑运算符
– orders List 否 排序1、复合指标与同环比类函数指标不支持排序2、定义驱动当前不支持按日期、维度进行排序。
— ascending boolean 否 升降序,true 正序，false 倒序
— propertyName String 否 排序属性名
– aliass List 否 （仅需调用方关注）字段别名(节后升级为aliases)
— aliasName String 否 字段别名
— name String 否 字段名
– maxResults Integer 否 查询条数，1~2000
– firstResult Integer 否 查询数据起始位置 第一条为0
– group List 否 聚合维度
